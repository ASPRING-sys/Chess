国际象棋联机对战系统 (Chess Online)
项目简介
本项目是一个基于 C++ 开发的异步联机国际象棋系统。系统采用后端引擎与前端网页分离的架构，支持多人在线建房、对弈、观战以及实时聊天。后端使用 Crow 框架处理 WebSocket 通信，前端利用 chessboard.js 进行棋盘渲染与交互。

核心功能
1. 完整的国际象棋规则
基础走子逻辑：涵盖所有棋子的合法移动判定。

特殊着法：支持“吃过路兵”（En Passant）、“王车易位”以及“兵升变”（Pawn Promotion）。

胜负判定：具备将军（Check）、将死（Checkmate）、僵局（Stalemate）及和棋判定。

2. 联机房间系统
建房机制：玩家可以创建房间并选择偏好颜色（执白或执黑）。

加入房间：通过唯一的 6 位房间代码（如 7B2K9X）进入特定对局。

角色分配：系统自动识别房主、挑战者及观战者身份。

房主权限：房主拥有管理权限，可以指定对手或将违规玩家踢出房间。

3. 通信与同步
状态同步：基于 FEN (Forsyth-Edwards Notation) 字符串实时同步全场棋局状态。

WebSocket 通信：前端与后端通过 JSON 格式消息进行低延迟双向交互。

实时聊天：对局双方及观战者可在聊天框进行实时交流。

4. 容错与恢复机制
断线重连：记录玩家唯一 ID (UID)，支持玩家在刷新页面或意外掉线后重新进入房间并恢复角色身份。

房间回收：当房间全员离开时进入 15 秒回收倒计时，若期间无人重连则自动销毁资源。

状态保护：非法走子由后端拦截，确保棋局逻辑的绝对严谨。

技术栈
后端 (C++)
开发语言：C++ 11/14

网络框架：Crow (基于 Asio)

协议：WebSocket, HTTP

序列化：JSON (Crow内置)

前端 (Web)
基础：HTML5, CSS3, JavaScript

棋盘库：chessboard.js

交互库：jQuery

样式：原生 CSS (移动端适配)

项目结构
main.cpp: 后端核心逻辑，负责服务器启动、房间管理及消息路由。

game.h / game.cpp: 游戏引擎类，处理棋盘状态记录及胜负判定。

pieces.h / pieces.cpp: 棋子基类及派生类，定义多态的移动规则。

pawn.h: 专门处理兵的复杂逻辑（升变、过路兵）。

room.h: 房间对象类，维护玩家连接及游戏实例。

index.html: 前端单一入口文件，包含 UI 布局及 WebSocket 客户端逻辑。

运行环境要求
支持 C++11 或更高版本的编译器 (GCC / MSVC)。

需配置 Asio 库。

确保 8080 端口未被占用（或在 main.cpp 中自行修改）。
